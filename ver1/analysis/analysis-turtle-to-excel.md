# Анализ конвертера «turtle to excel»

Анализ реализации преобразования Turtle RDF → Excel (вкладка «Turtle → Excel»,
модули [`turtle.js`](../js/turtle.js), [`megatype.js`](../js/megatype.js),
[`workbook.js`](../js/workbook.js)).

## Назначение

Превратить RDF-граф (Turtle) в книгу Excel, удобную для просмотра, фильтрации,
сортировки и ручной правки, с возможностью обратной выгрузки в Turtle.

## Конвейер обработки

1. **Разбор Turtle** (`parseTurtle`) → модель `{ prefixes, triples, base }`.
   Триплеты хранятся в исходной форме термов (префиксное имя / IRI / литерал).
2. **Построение МегаТипов** (`buildMegatypes`) — группировка субъектов по значению
   предиката `:МегаТип`, сбор предикатов-столбцов и значений.
3. **Раскладка по листам** (`modelToSheets`) — формирование «массивов массивов» (AOA):
   служебные листы + по листу на каждый МегаТип + «Прочие триплеты».
4. **Сборка книги** (`modelToWorkbook` через SheetJS) и скачивание `.xlsx`.

## Сильные стороны

- **Полностью браузерный код**, без node.js; запуск на GitHub Pages (требование проекта).
- **Обратимость (round-trip)** — данные раскладываются так, что Excel → Turtle
  восстанавливает исходный набор триплетов (покрыто тестом).
- **Читаемость** — термы сохраняются в префиксной форме, листы названы по МегаТипам.
- **Отделение чистой логики** (раскладка AOA) от ввода-вывода (XLSX), что упрощает
  тестирование без зависимости от библиотеки таблиц.
- **Устойчивость к «лишним» данным** — субъекты без МегаТипа не теряются (лист
  «Прочие триплеты»).

## Ограничения и риски

| Тема | Ограничение | Возможное развитие |
|------|-------------|--------------------|
| Полнота Turtle | Парсер поддерживает основные конструкции (префиксы, `;`, `,`, `a`, литералы с `@lang`/`^^`, комментарии). Blank nodes `[]` и коллекции `()` распознаются как разделители, но не разворачиваются в полноценную структуру | Добавить поддержку blank nodes и коллекций |
| Множественный МегаТип | Предполагается один МегаТип на субъект; при нескольких берётся последний | Выбор основного МегаТипа или дублирование на нескольких листах |
| Имена листов | Excel ограничивает имя листа 31 символом и набором символов; длинные/совпадающие локальные имена усекаются и нумеруются | Хранить полное имя МегаТипа в отдельном столбце листа `main` |
| Типы данных литералов | Литералы переносятся как текст «как есть» (с кавычками) | Раскладывать значение/язык/тип в отдельные столбцы |
| Очень большие графы | Вся обработка в памяти браузера | Потоковая обработка / постраничность |

## Производительность

Сложность линейна по числу триплетов: один проход для разбора, один — для группировки,
один — для раскладки. Для типичных моделей (сотни–тысячи триплетов) время в браузере
незаметно. Основные затраты — генерация `.xlsx` библиотекой SheetJS.

## Тестируемость

Чистые функции (`parseTurtle`, `buildMegatypes`, `modelToSheets`, `sheetsToModel`)
тестируются в node без XLSX. Полный файловый round-trip с реальным `.xlsx` проверяется
дополнительно (см. процесс в [`tests/`](../tests)).

## Вывод

Подход «property table по МегаТипу» даёт наглядное и обратимое представление RDF в Excel,
хорошо подходит для ручного редактирования предметных моделей (процессы, роли,
оргструктуры). Ограничения касаются редких конструкций Turtle (blank nodes, коллекции)
и крайних случаев именования листов; они не мешают основному сценарию и могут быть сняты
итеративно.
