# Описание проверок VBA (валидация листов МегаТипов)

Этот файл описывает все проверки, реализованные в модуле
[`ver3/vba/Validation.bas`](../vba/Validation.bas). Проверки помогают пользователю
редактировать книгу Excel в удобном табличном виде и при этом сохранять корректность
данных перед выгрузкой обратно в Turtle RDF.

## Когда срабатывают проверки

Проверки вызываются автоматически при изменении ячейки на листе МегаТипа — через
событие `Worksheet_Change`, которое перенаправляется в `modValidation.ValidateChange`.
Часть проверок (например, соответствие столбца `:МегаТип` имени листа) можно запустить
для всего листа вручную кнопкой, привязанной к `ValidateSheetButton`.

Служебные листы (`main`, `Префиксы`, `Turtle исходный`, `Триплеты простые`,
`Триплеты компактные`, `Прочие триплеты`) не проверяются — проверки относятся только
к листам МегаТипов.

## Перечень проверок

| № | Проверка | Когда | Реакция | Процедура |
|---|----------|-------|---------|-----------|
| 1 | **Дубликат предиката** | Изменён заголовок столбца (строка 1) | Предупреждение, если такой предикат (заголовок) уже есть на листе | `CheckDuplicatePredicate` |
| 2 | **Пустой субъект** | Изменён первый столбец «Субъект» | Предупреждение, если в строке есть значения, но субъект пуст | `CheckEmptySubject` |
| 3 | **Дубликат субъекта** | Изменён первый столбец «Субъект» | Предупреждение, если такой субъект уже встречается в столбце (объекты одного субъекта надо перечислять через запятую в одной строке) | `CheckDuplicateSubject` |
| 4 | **Непарные кавычки в литерале** | Изменена ячейка-значение | Предупреждение, если число кавычек `"` нечётное | `CheckTermSanity` |
| 5 | **Незакрытый IRI** | Изменена ячейка-значение | Предупреждение, если терм начинается с `<`, но не заканчивается на `>` | `CheckTermSanity` |
| 6 | **Подозрительный терм** | Изменена ячейка-значение | Подсказка, если значение не похоже ни на префиксное имя (`prefix:local`), ни на IRI (`<...>`), ни на литерал (`"..."`) | `CheckTermSanity` |
| 7 | **Пробел в терме** | Изменена ячейка-значение | Подсказка: вероятно, нужен литерал в кавычках или разделение объектов запятой | `CheckTermSanity` |
| 8 | **Несоответствие МегаТипа** | Запуск проверки листа вручную | Перечисляет строки, где значение в столбце `:МегаТип` не совпадает с именем листа (МегаТипом) | `ValidateSheet` |

## Подробности

### 1. Дубликат предиката (столбца)
По концепции проекта пользователь может добавлять предикаты (столбцы) вручную.
Если добавляемый предикат уже присутствует на листе как заголовок другого столбца,
выводится предупреждение — это устраняет двусмысленность при обратном преобразовании
в Turtle.

### 2. Пустой субъект
Каждая строка листа МегаТипа описывает один объект (субъект). Если в строке заполнены
значения предикатов, но первый столбец «Субъект» пуст, триплеты построить невозможно —
выводится предупреждение.

### 3. Дубликат субъекта
Один субъект должен занимать одну строку. Несколько объектов одного предиката
указываются в одной ячейке через запятую (`:A , :B`), а несколько предикатов —
в разных столбцах. Повтор субъекта в нескольких строках, скорее всего, ошибка ввода.

### 4–7. Корректность терма
Значение ячейки может содержать несколько объектов, разделённых запятой. Каждый объект
проверяется отдельно:
- литерал — должен иметь парные кавычки `"..."`;
- IRI — должен быть заключён в `<...>`;
- префиксное имя — вида `prefix:local`, без пробелов;
- значение без `:`, кавычек и угловых скобок (и не ключевое слово `a`) помечается как
  подозрительное.

### 8. Соответствие столбца `:МегаТип` имени листа
Лист МегаТипа содержит объекты одного МегаТипа. Значение в столбце `:МегаТип` каждой
строки (например, `:Процесс`) по локальному имени должно совпадать с именем листа
(`Процесс`). Несоответствия перечисляются с указанием номеров строк.

## Соответствие проверок логике конвертера

Те же по смыслу инварианты, что проверяет VBA в Excel, конвертер `excel → turtle`
(файл [`ver3/js/workbook.js`](../js/workbook.js)) учитывает при сборке триплетов:
пустые ячейки пропускаются, значения с запятыми разбиваются на несколько объектов,
пустые субъекты игнорируются. Таким образом, корректно отредактированная книга
без предупреждений даёт корректный Turtle RDF.
