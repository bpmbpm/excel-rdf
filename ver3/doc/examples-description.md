# Описание примеров

Примеры лежат в каталоге [`ver3/examples/`](../examples). Файлы `.ttl` можно загрузить во
вкладке «Turtle → Excel», а файлы `.trig` — во вкладке «TriG → Excel», и преобразовать в книгу Excel.

## example1.ttl — процессы и роли

Простая модель: объекты двух МегаТипов — `:Процесс` и `:Роль`.

```turtle
:Процесс_А :МегаТип :Процесс ;
    rdfs:label "Согласование договора" ;
    :владелец :Иванов ;
    :входит_в :Процесс_Б .
```

Что показывает:
- группировку субъектов по МегаТипу в отдельные листы («Процесс», «Роль»);
- предикат-ссылку на другой объект (`:владелец :Иванов`, `:входит_в :Процесс_Б`);
- литералы (`rdfs:label "..."`).

В книге появятся листы: `main`, `Префиксы`, `Turtle исходный`, `Триплеты простые`,
`Триплеты компактные`, `Процесс`, `Роль`.

## example2.ttl — несколько объектов на предикат

Модель организации: подразделения и сотрудники. Демонстрирует **списки объектов**
через запятую и несколько предикатов через точку с запятой.

```turtle
:Отдел_Продаж :МегаТип :Подразделение ;
    rdfs:label "Отдел продаж" ;
    :включает :Сидоров , :Кузнецов , :Смирнов .
```

Что показывает:
- несколько объектов одного предиката (`:включает :Сидоров , :Кузнецов , :Смирнов`) —
  в ячейке Excel они записываются через запятую;
- два МегаТипа: `:Подразделение` и `:Сотрудник`.

## example3.ttl — объекты с МегаТипом и «прочие» триплеты

Показывает, что триплеты субъектов **без** `:МегаТип` (например, описание класса
`:Товар a owl:Class`) попадают на отдельный лист **«Прочие триплеты»**, а не теряются.

```turtle
:Товар_1 :МегаТип :Товар ;
    rdfs:label "Ноутбук" ;
    :цена "75000" ;
    :категория :Электроника .

:Товар a owl:Class ;
    rdfs:label "Товар" .
```

Что показывает:
- объекты МегаТипа `:Товар` — на листе «Товар»;
- субъект `:Товар` (без `:МегаТип`) — на листе «Прочие триплеты»;
- сохранение ключевого слова `a` (rdf:type).

## example4.ttl — детальный пример

Расширенный пример с большим числом предикатов (введён в ver2).
Содержит четыре МегаТипа — `:Процесс`, `:Роль`, `:Документ`, `:Система` — причём у
МегаТипа `:Процесс` девять предикатов (`rdfs:label`, `dcterms:description`, `:владелец`,
`:входныеДанные`, `:выходныеДанные`, `:использует`, `:следующий`, `:длительность`,
`:приоритет`).

```turtle
:Процесс_Согласование :МегаТип :Процесс ;
    rdfs:label "Согласование договора" ;
    dcterms:description "Первый процесс маршрута" ;
    :владелец :Менеджер ;
    :использует :CRM , :Хранилище ;
    :следующий :Процесс_Подписание .
```

Что показывает:
- богатый набор предикатов на одном листе МегаТипа — наглядно работает столбец
  «Предикаты» на листе `main` (требование #3): можно сравнить наборы полей разных МегаТипов;
- несколько объектов одного предиката (`:использует :CRM , :Хранилище`);
- запятые внутри литералов (`dcterms:description "Второй процесс, выполняется после Процесса 1"`)
  не считаются разделителями объектов.

## trig-vad.trig — именованные графы и МегаТипы (подробный пример TriG)

Пример TriG для вкладки «TriG → Excel». Взят из
[rdf-grapher/ver9d/dia/Trig_VADv8.ttl](https://github.com/bpmbpm/rdf-grapher/blob/main/ver9d/dia/Trig_VADv8.ttl).
Содержит именованные графы `vad:root`, `vad:ptree`, `vad:rtree`, `vad:t_p1`, `vad:t_p1_1`,
`vad:t_p2`. В ver3 МегаТипами (по `rdf:type`) выступают `vad:ObjectTree`, `vad:TypeProcess`,
`vad:TypeExecutor`, `vad:VADProcessDia`, `vad:ExecutorGroup`; корень дерева исправлен на
`vad:root rdf:type vad:ObjectTree`.

```trig
vad:ptree {
    vad:p1 rdf:type vad:TypeProcess ;
        rdfs:label "p1 Процесс 1" ;
        vad:hasParentObj vad:ptree .
}
```

Что показывает (в раскладке ver3):
- раскладку субъектов по листам **МегаТипов** (по `rdf:type`), а не по графам;
- столбец «TriG» на листе МегаТипа — именованный граф каждого утверждения;
- общий лист «TriG» (свойства графов) и реестр «МегаТипы»;
- листы «TriG с Триплеты простые» и «TriG с Триплеты компактные»;
- полный round-trip TriG → Excel → TriG без потери квадров.

## trig-simple.trig — простой пример TriG (второй пример)

Более простой пример TriG (требование #4 issue #5) — книга-предприятие из двух именованных
графов: `ex:catalog` (каталог товаров) и `ex:orders` (заказы). МегаТипы: `ex:Product`,
`ex:Customer`, `ex:Order`.

```trig
ex:catalog {
    ex:apple rdf:type ex:Product ;
        rdfs:label "Яблоко" ;
        ex:price "30" .
}

ex:orders {
    ex:order1 rdf:type ex:Order ;
        ex:item ex:apple , ex:bread .
}
```

Что показывает:
- небольшой набор данных, на котором легко увидеть листы МегаТипов и столбец «TriG»;
- несколько объектов одного предиката (`ex:item ex:apple , ex:bread`);
- один и тот же субъект встречается в разных графах — отдельной строкой на каждый граф.

## Проверка round-trip

Все примеры покрыты автоматическим тестом
[`ver3/tests/test-ver3.js`](../tests/test-ver3.js): Turtle → листы книги → Turtle и
TriG → листы книги → TriG не теряют триплетов/квадров и префиксов; проверяется также
наличие закрепления строки и автофильтра в реальном `.xlsx`. Запуск (только для разработки, node):

```
node ver3/tests/test-ver3.js
```
