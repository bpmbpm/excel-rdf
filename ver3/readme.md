# ver2 — конвертеры Turtle RDF ⇄ Excel и TriG ⇄ Excel

Браузерное приложение (только JavaScript, без node.js; запуск на GitHub Pages),
выполняющее четыре преобразования:

- **Turtle RDF → Excel** — раскладывает RDF-граф по листам книги (концепция «МегаТип»);
- **Excel → Turtle RDF** — собирает корректный Turtle из отредактированной книги;
- **TriG → Excel** — раскладывает именованные графы TriG по листам (один лист = один граф);
- **Excel → TriG** — собирает корректный TriG из отредактированной книги.

## Что нового в ver2 (относительно ver1)

1. **Закрепление первой строки и автофильтр на каждом листе** любой создаваемой книги.
2. **Детальный пример** с большим числом предикатов — [`examples/example4.ttl`](examples/example4.ttl).
3. **Столбец «Предикаты» на листе `main`** — для каждого листа МегаТипа/графа перечислены
   его предикаты, наборы полей удобно сравнивать в одном месте.
4. **Конвертеры TriG ⇄ Excel** с поддержкой именованных графов.
5. **Каталог `ver2/`** с обновлённой документацией.

## Запуск

- Онлайн: `https://bpmbpm.github.io/excel-rdf/ver2/`
- Локально: откройте [`index.html`](index.html) в браузере (для кнопок «Загрузить пример»
  нужен запуск через локальный веб-сервер, например `python3 -m http.server`).

## Документация

Начните с [`doc/readme.md`](doc/readme.md): состав книги Excel (Turtle и TriG),
концепция «МегаТип», закрепление строки и автофильтр, конвертеры TriG, обратное
преобразование, проверки VBA и скриншоты.

## Тесты (для разработки)

Логика конвертеров покрыта тестами на чистых функциях (раскладка по листам и обратная
сборка не зависят от библиотеки таблиц); при наличии SheetJS дополнительно проверяется
реальный `.xlsx` (закрепление строки, автофильтр, round-trip):

```
node ver2/tests/test-ver2.js
```

> Примечание. В готовом продукте используется только браузерный JavaScript, а таблицы
> создаёт библиотека [SheetJS](https://sheetjs.com/), подключаемая из CDN. Каталог
> `tests/` и node нужны только для разработки.
