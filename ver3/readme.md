# ver3 — конвертеры Turtle RDF ⇄ Excel и TriG ⇄ Excel

Браузерное приложение (только JavaScript, без node.js; запуск на GitHub Pages),
выполняющее четыре преобразования:

- **Turtle RDF → Excel** — раскладывает RDF-граф по листам книги (концепция «МегаТип»);
- **Excel → Turtle RDF** — собирает корректный Turtle из отредактированной книги;
- **TriG → Excel** — раскладывает утверждения TriG по листам **МегаТипов** (по `rdf:type`),
  принадлежность к именованному графу указывается в столбце «TriG»;
- **Excel → TriG** — собирает корректный TriG из отредактированной книги.

## Что нового в ver3 (относительно ver2)

Главное изменение ver3 (см. [issue #5](https://github.com/bpmbpm/excel-rdf/issues/5)) —
листы анализа утверждений TriG строятся вокруг **МегаТипов**, а не вокруг графов.

1. **Листы TriG по МегаТипам.** В ver2 один лист = один именованный граф. В ver3 один
   лист = один МегаТип (значение `rdf:type` субъекта; `a` и `rdf:type` равнозначны).
   На каждом листе МегаТипа добавлено поле **«TriG»**, указывающее, к какому именованному
   графу относится утверждение. Строка листа — пара (субъект, граф).
2. **Общий лист «TriG»** — сводка по каждому именованному графу: его субъекты, свойства
   (предикаты) и число квадров.
3. **Реестр «МегаТипы»** — перечень листов МегаТипов и их термов `rdf:type` (используется
   при обратной сборке Excel → TriG).
4. **Текстовые листы форм TriG**: «TriG с Триплеты простые» (внутри графов по одному
   триплету на строку, без `;` и `,`) и «TriG с Триплеты компактные» (с `;` и `,`).
   Аналог листов «Триплеты простые/компактные» книги Turtle.
5. **Второй, более простой пример TriG** — [`examples/trig-simple.trig`](examples/trig-simple.trig)
   (книга-предприятие из двух графов `ex:catalog` и `ex:orders`).
6. **Пример `trig-vad.trig` и онтология приведены к МегаТипам.** В качестве МегаТипов заданы
   `vad:ObjectTree`, `vad:TypeProcess`, `vad:TypeExecutor`, `vad:VADProcessDia`,
   `vad:ExecutorGroup`; корень исправлен на `vad:root rdf:type vad:ObjectTree`. В онтологии
   [`ontology/vad-basic-ontology.trig`](ontology/vad-basic-ontology.trig) добавлен класс-маркер
   `vad:MegaType`, которым помечены пять классов-МегаТипов.

Сохранена обратимость (round-trip): `rdf:type` хранится как обычный столбец листа МегаТипа,
поэтому TriG → Excel → TriG не теряет и не выдумывает квадров.

## Запуск

- Онлайн: `https://bpmbpm.github.io/excel-rdf/ver3/`
- Локально: откройте [`index.html`](index.html) в браузере (для кнопок «Загрузить пример»
  нужен запуск через локальный веб-сервер, например `python3 -m http.server`).

## Документация

Начните с [`doc/readme.md`](doc/readme.md): состав книги Excel (Turtle и TriG ver3),
концепция «МегаТип», закрепление строки и автофильтр, конвертеры TriG, обратное
преобразование, проверки VBA и скриншоты.

## Тесты (для разработки)

Логика конвертеров покрыта тестами на чистых функциях (раскладка по листам и обратная
сборка не зависят от библиотеки таблиц); при наличии SheetJS дополнительно проверяется
реальный `.xlsx` (закрепление строки, автофильтр, round-trip):

```
node ver3/tests/test-ver3.js
```

> Примечание. В готовом продукте используется только браузерный JavaScript, а таблицы
> создаёт библиотека [SheetJS](https://sheetjs.com/), подключаемая из CDN. Каталог
> `tests/` и node нужны только для разработки.
