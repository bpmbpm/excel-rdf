# Описание проекта в терминах Linked Data

Документ описывает проект excel-rdf в принятых в Linked Data / RDF терминах.

## RDF и Turtle

**RDF (Resource Description Framework)** — модель данных, в которой информация
представляется в виде **триплетов**: `субъект — предикат — объект`
(`subject — predicate — object`).

- **Субъект** (subject) — ресурс, о котором делается утверждение (IRI или blank node).
- **Предикат** (predicate) — свойство/отношение (всегда IRI).
- **Объект** (object) — значение свойства: другой ресурс (IRI), blank node или
  литерал (literal) — строка, число, дата и т.п.

**Turtle (Terse RDF Triple Language)** — текстовый синтаксис RDF. Ключевые элементы,
которые понимает конвертер:

| Элемент Turtle | Назначение |
|----------------|------------|
| `@prefix ex: <IRI> .` | Объявление префикса для сокращения IRI |
| `ex:foo` | Префиксное имя (CURIE) — раскрывается в полный IRI |
| `<http://...>` | Полный IRI |
| `"текст"`, `"текст"@ru`, `"5"^^xsd:int` | Литералы (со языковым тегом или типом данных) |
| `a` | Сокращение для `rdf:type` |
| `;` | Тот же субъект, следующий предикат |
| `,` | Тот же субъект и предикат, следующий объект |
| `.` | Конец утверждения |

## Граф и его проекция в таблицы

RDF — это **граф**: множество триплетов образует направленные рёбра между узлами.
Таблица (лист Excel) — двумерная структура, поэтому проект применяет проекцию графа
в таблицы по принципу «**объектно-ориентированного среза**»:

- узлы-субъекты одного **типа** становятся **строками** одной таблицы;
- предикаты этого типа становятся **столбцами**;
- объекты (значения) располагаются на пересечении.

Это близко к понятию **property table** (таблица свойств), известному в хранилищах RDF,
и к форме **wide table** в реляционных проекциях RDF.

## МегаТип как типизирующее свойство

В классическом RDF тип ресурса задаётся через `rdf:type` (`a`). В данном проекте
вводится отдельное типизирующее свойство `:МегаТип`:

```turtle
:Процесс_А :МегаТип :Процесс .
```

С точки зрения Linked Data `:МегаТип` — это обычный предикат, играющий роль
**классифицирующего свойства** (аналог `rdf:type`), по значению которого выполняется
разбиение графа на **классы эквивалентности субъектов**. Каждому значению `:МегаТип`
(классу) соответствует отдельная property-table (лист Excel).

Отличие от `rdf:type`:
- `:МегаТип` задаётся пользователем предметной области и отражает крупную категорию
  («МегаТип» — мета-тип, верхнеуровневая классификация: Процесс, Роль, Подразделение …);
- один субъект имеет ровно один лист (один основной МегаТип), что обеспечивает
  однозначную табличную раскладку.

## Открытость мира и неполнота строк

В RDF действует **допущение открытого мира** (Open World Assumption): отсутствие
триплета не означает ложность. В табличной проекции это проявляется как **пустые
ячейки**: если у объекта нет некоторого предиката, ячейка пуста, и при обратном
преобразовании Excel → Turtle такой триплет просто не создаётся (а не превращается
в «пустое» утверждение).

## Сохранение идентификаторов (IRI) и префиксов

Чтобы данные оставались **связанными** (Linked), конвертер сохраняет:
- объявления префиксов (лист «Префиксы») — отображение `prefix → namespace IRI`;
- термы в исходной форме (префиксное имя, полный IRI или литерал) — без принудительного
  раскрытия в полный IRI, что сохраняет читаемость и обратимость.

Вспомогательные функции `localName()` и `expandTerm()` (см.
[`ver2/js/turtle.js`](../js/turtle.js)) позволяют при необходимости получить локальное
имя или полный IRI терма.

## Round-trip и сохранение семантики

Главное требование к связыванию данных — **обратимость** (round-trip): Turtle → Excel →
Turtle не должно терять и искажать триплеты. В проекте это обеспечивается тем, что
множество триплетов из листов МегаТипов и листа «Прочие триплеты» совпадает с исходным
(проверяется автоматическим тестом [`tests/test-ver2.js`](../tests/test-ver2.js)).
Порядок триплетов и форма группировки (`;`, `,`) могут отличаться, но это
семантически эквивалентные сериализации одного и того же RDF-графа.

## TriG и именованные графы (ver2)

**TriG** расширяет RDF/Turtle понятием **именованного графа**: тройка превращается в
**четвёрку (quad)** `граф — субъект — предикат — объект`. Синтаксически триплеты
группируются в блоки `graphLabel { ... }`; триплеты вне блоков относятся к **графу по
умолчанию** (default graph).

| Элемент TriG | Назначение |
|--------------|------------|
| `:g { :s :p :o . }` | Триплеты, помещённые в именованный граф `:g` |
| `:s :p :o .` (вне скобок) | Триплет графа по умолчанию |
| `GRAPH :g { ... }` | То же, что `:g { ... }` (необязательное ключевое слово) |

Это соответствует модели **RDF Dataset**: набор именованных графов плюс один безымянный
граф по умолчанию. В табличной проекции ver2 каждый именованный граф становится отдельной
property-table (лист Excel), а соответствие «лист ⇄ метка графа» хранится на листе «Графы».
Полный список четвёрок дублируется на листе «Квадры простые». Round-trip TriG → Excel →
TriG сохраняет все квадры и префиксы (см. тест [`tests/test-ver2.js`](../tests/test-ver2.js)).

| Термин проекта (TriG) | Термин Linked Data / RDF |
|-----------------------|--------------------------|
| Граф (метка) | Имя именованного графа (named graph IRI) |
| Лист графа | Property table одного именованного графа |
| Квадра | Quad / statement в составе RDF Dataset |
| Граф по умолчанию | Default graph (триплеты без метки) |

## Терминологическое соответствие

| Термин проекта | Термин Linked Data / RDF |
|----------------|--------------------------|
| Объект (строка листа) | Субъект-ресурс (subject, IRI) |
| Поле / столбец | Предикат (predicate, property) |
| Значение ячейки | Объект триплета (object: IRI или литерал) |
| МегаТип | Классифицирующее свойство (аналог `rdf:type`), класс субъектов |
| Лист МегаТипа | Property table (таблица свойств) для одного класса |
| Префикс | Namespace prefix / CURIE |
| Триплет (простой лист) | Statement `s p o` |
| Компактная форма (`;`, `,`) | Predicate-object list / object list |
